# BIT3208--Advanced-Web-Development
KikapuMart is a web-based e-commerce system developed using PHP, MySQL, HTML, and CSS to provide an online marketplace where customers can browse products, manage shopping carts, and place orders. The system was designed to simplify online shopping by providing an easy-to-use interface for customers while also offering administrators a secure platform for managing products, customers, and orders. The project utilizes Apache through XAMPP/LAMPP as the web server and phpMyAdmin for database management.

The primary objective of KikapuMart is to provide a complete online shopping solution that enables customers to register accounts, log in securely, browse available products, add products to a shopping cart, proceed through the checkout process, and place orders. At the same time, the system provides administrative functionalities that allow administrators to monitor customer activities, manage inventory, view orders, and maintain product information. This ensures efficient business operations and improved customer service.

The customer module of the system includes features such as customer registration, login authentication, product browsing, shopping cart management, checkout processing, order placement, and logout functionality. These features allow users to create personal accounts and perform online shopping activities conveniently. The administrator module includes a secure login system, an administrative dashboard, product management tools, customer management functions, order monitoring capabilities, and logout functionality. Access to administrative pages is restricted to authorized administrators only.

The database used by the system is named KikapuMartdb and contains several related tables. The customers table stores customer information including customer identification numbers, names, email addresses, passwords, phone numbers, addresses, and registration dates. The products table contains product details such as product names, categories, descriptions, prices, stock quantities, brands, and image references. The orders table records customer purchases by storing order identifiers, customer identifiers, order dates, total amounts, order statuses, delivery addresses, and payment methods. The order_items table stores detailed information about products included in each order, including quantities and prices. An admin table is used to store administrator login credentials for secure access to the management section of the system.

The project is organized into several PHP and HTML files that work together to provide the required functionality. The main files include index.php for the homepage, products.php for displaying products, cart.php for shopping cart management, checkout.php for collecting delivery information, place_order.php for processing customer orders, login.php and register.php for customer authentication, and logout.php for ending user sessions. Additional administrative files are stored in a dedicated admin directory and include pages for administrator login, dashboard management, product management, order viewing, customer viewing, and administrator logout.

To install and run the system, XAMPP or LAMPP must first be installed and configured with Apache and MySQL services running. The KikapuMart project folder should then be copied into the htdocs directory. A MySQL database named KikapuMartdb should be created through phpMyAdmin, and the required database tables should be imported. The database connection settings must be configured in the db_connect.php file to match the local server settings. Once setup is complete, the customer portal can be accessed through the localhost URL, while administrators can access the administrative portal through the admin login page.
